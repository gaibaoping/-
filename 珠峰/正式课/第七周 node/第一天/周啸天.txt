    3、扫盲（DOS命令操作）
        1)查看自己电脑的IP地址
            ipconfig 查看自己每一个网络的IP地址
                Ipv4的地址：192.168.0.20
                子网掩码：255.255.255.0
                默认网关：192.168.0.1
            如果连接的是同一个网络（WIFI），并且子网掩码和默认网关都一样，一般情况下说明两台设备在同一个局域网下：假设我的电脑和我的手机在同一个局域网下，这样我只需要在自己的电脑上发布一个小程序，手机上就可以访问了（移动端开发的真机测试可以这样搞）

            ipconfig -all 不仅可以查看IP，还可以查看本机的MAC地址（物理地址）:当我们的设备连接网络后，会根据IP以及我们电脑的本身生成一个全世界唯一的字符串，这个就是物理地址
        2)查看自己的网速
            ping www.baidu.com
            ping www.baidu.com -t
        3)结束之前正在运行的DOS操作
            control 加c

            退出当前的DOS窗口  exit

            清屏  cls
        4)打开DOS命令后，我们的操作默认的是在一个指定的文件夹下执行的，这样的话我们只能操作指定文件夹下的内容，例如：C:\User\zhufengpeixun\
        有时候我们需要进入指定的文件夹之下操作，例如：D:\shareProject\animate\src

        D:直接进入到D盘（不管当前在哪个文件下，直接输入磁盘号，快速进入到某一个磁盘）
        D:cd b(进入到指定的文件夹)
            dir 查看当前目录下所有的文件
            mkdir 文件夹名称 在当前的目录下新创建文件夹
            copy con xxx.txt ->开始编辑文件中的内容 ->CTRL+Z结束内容的编辑即可在指定的目录下新建文件，以及向文件中编辑内容

        cd ../ 返回到当前目录的上一级目录
        cd / 返回到当前目录的根目录
        cd ./ 返回到当前目录所在的目录

    4、
        面试题：当用户在自己的浏览器地址栏输入一个网址，到看到一个完整的HTML页面，中间都经历了哪些事情？

        ->反推：想让用户看到我们的网页，我们需要做哪些事情？
            1)买一台服务器（虚拟服务器/云空间/云服务器 建议：阿里云服务器），服务器连接上网络后，会存在一个供外面访问的IP地址（外网IP：180.76.161.28）
            2)把自己的项目文件上传到服务器上（通过FTP上传工具上传）
            3)买一个域名（万网->被阿里收购了）www.zhufengpeixun.cn
            4)DNS解析：在DNS服务器（全世界网络管理服务费）上把我们的域名和服务器的外网IP地址关联到一起
            5)发布服务，给当前的服务监听一个端口号（而且当前服务的端口号不能和其他服务的重复）：当客户端向服务器发送请求了，找到当前项目负责的服务，由这个服务来完成一些操作。

    5、访问服务器
        1、通过浏览器地址栏中的域名，到DNS服务器上找到对应服务器的外网IP地址
        2、通过外网IP找到对应的服务器
        3、通过地址栏中输入的端口号找到对应的服务。
        4、找到的服务把需要的资源文件中的源代码返回给客户即可
        5、客户拿到需要的源代码之后，客户端的浏览器会把源代码解析为一个网页来呈现

    ->总结：
    1)一台服务器上可以发布好几个项目，每一个项目都有一个服务来负责，每一个服务都有一个自己对应的端口号，不同的服务之间端口号不能重复。每一台服务器端口号都是有范围的：0~65535之间

    2)传输协议：客户端和服务器端内容或者数据的传输都是通过传输协议来完成的（或者把传输协议理解为快递小哥即可）
        HTTP：超文本传输协议，除了传递文字文本内容以外，还可以传递图片音视频等内容。
        HTTPS：HTTP SSL(经过加密的HTTP，比传统的HTTP传输更加安全)
        FTP：资源文件传输协议，一般用于把项目资源文件上传到服务器的指定磁盘位置
        ...

        如果没有在URL地址栏中指定端口，那么每一个传输协议都会默认找对应的端口来为客户端服务
        ->HTTP  默认找 80
        ->HTTPS  默认找 443
        ->FTP  默认找 21

    3)域名
        一级域名 www.qq.com
        二级域名 sports.qq.com
        三级域名 kbs.sports.qq.com
        但是他们所属的域是一样的，都是qq.com这个域，或句话说也就是 document.domain = 'qq.com'

    4)URI/URL/URN
        URI：统一资源标识符 URL+URN
        URL：统一资源定位符
        URN：统一资源名称

        URL：地址
        http://www.zhufengpeixun.cn:80/css/index.css?name=zf&age=7#video

        /*
        * css/index.css:客户端请求资源文件的路径和名称（告诉服务端需要请求内容的目录）
        * ?name=zf&age=7:问号传参（客户端传递给服务器端的内容）
        * #vido:锚点定位（定位到当前页面指定ID的区域）/HASH值（哈希值）
        */

    5)HTTP事物
        事物就是一件完整的事情
            ->客户端可以传递一些内容给服务器端（Request请求阶段）
            ->服务器端把客户端需要的内容返回（Response响应阶段）
            客户端和服务器端的内容传输都是通过传输协议完成的（一般传输协议都是HTTP）
        HTTP事物 = Request+Response

    6)代码解析
        W3C：一个非盈利机构，用来制定编程规范和标准的地方
        （HTML/HTML5/CSS/CSS3/ECMAScript...）
        浏览器开发商按照标准和规范开发出浏览器（浏览器有一个东西叫引擎（内核））
        我们前端开发者按照标准写出代码
        我们把代码在浏览器的引擎中运行，浏览器的引擎按照标准把我们的代码渲染成页面和对应的样式效果

        IE内核（引擎Trident）：IE5~11、IE、Edge、360、猎豹、百度...
        Gecko内核：火狐
        Webkit内核（引擎V8）：谷歌、safari、QQ
        Presto内核：Opera 欧朋浏览器
